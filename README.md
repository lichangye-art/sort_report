## 一、 前言
在資訊科學領域中，排序是資料處理最基礎且核心的環節。選擇合適的排序演算法，不僅可以優化系統的執行效率，更能有效節省運算資源。此報告將針對氣泡、選擇、插入、合併及堆積五種演算法進行深入探討。

---

## 二、 演算法原理與複雜度分析

### 1. 氣泡排序法 (Bubble Sort)
**原理簡述：**
氣泡排序是一種直觀的交換排序法。它重複地走訪要排序的數列，一次比較兩個相鄰的元素，如果順序錯誤就進行交換。重複此過程直到沒有任何元素需要交換。其特徵是每一輪排序後，該輪最大的元素會移動到數列的最末端。

**複雜度分析：**
- **平均/最差時間複雜度：** $O(n^2)$
- **最佳時間複雜度：** $O(n)$
- **空間複雜度：** $O(1)$

### 2. 選擇排序法 (Selection Sort)
**原理簡述：**
選擇排序的核心是「在未排序區域中尋找極值」。演算法會將數列分為「已排序」與「未排序」兩個部分。每一輪都會從未排序區域中找出最小值，並將其放置在已排序區域的末尾。與氣泡排序不同，選擇排序每一輪只進行一次交換。

**複雜度分析：**
- **平均/最差時間複雜度：** $O(n^2)$
- **最佳時間複雜度：** $O(n^2)$
- **空間複雜度：** $O(1)$

### 3. 插入排序法 (Insertion Sort)
**原理簡述：**
插入排序的原理為將數列分為已排序與未排序兩部分。逐一取回未排序區域的元素，在已排序區域中從後往前掃描，找到合適的位置後「插入」。此方法對於處理「幾乎已排好序」的資料具有極高的效率。

**複雜度分析：**
- **平均/最差時間複雜度：** $O(n^2)$
- **最佳時間複雜度：** $O(n)$
- **空間複雜度：** $O(1)$

### 4. 合併排序法 (Merge Sort)
**原理簡述：**
合併排序採用「分治法」的策略。先將數列不斷對半分切，直到每個子序列只剩一個元素；接著，再將這些有序的子序列兩兩合併，合併過程中確保新的序列依然有序，最終重組成完整的排序數列。

**複雜度分析：**
- **平均/最佳/最差時間複雜度：** $O(n \log n)$
- **空間複雜度：** $O(n)$

### 5. 堆積排序法 (Heap Sort)
**原理簡述：**
堆積排序利用了「堆積」這種半有序的資料結構。首先將原始數列建立成一個最大堆積，此時根節點即為最大值。接著將根節點與最後一個節點交換，並移除該最大值，隨後對剩餘結構進行調整使其重新符合堆積定義。重複此動作直到堆積清空。

**複雜度分析：**
- **平均/最佳/最差時間複雜度：** $O(n \log n)$
- **空間複雜度：** $O(1)$

## 三、 綜合比較與結論

這五種演算法各具優勢與限制：

- **基礎演算法 ($O(n^2)$ 級別)：** 氣泡、選擇與插入排序實作簡單，適合小規模資料。其中 **插入排序** 在處理局部有序資料時表現最為出色。
- **高效演算法 ($O(n \log n)$ 級別)：** 合併排序與堆積排序在處理大規模資料時更具效能優勢。**合併排序** 具有高度穩定性，而 **堆積排序** 則在空間利用上最為節省。

在實際應用中，通常會根據資料量大小、記憶體限制以及穩定性需求，靈活選擇最好的排序策略。

## 四、模擬實驗與實驗結果比較

輸入六種不同的數據量，比較五種排序演算法所需的排序時間。此實驗我利用C++進行排序，省略了各種動畫，僅有進行「排序」這個任務而已。

| 資料量（Ｎ） | 氣泡排序法  | 選擇排序法  | 插入排序法  | 合併排序法  | 堆積排序法 | 
|--------|--------|--------|--------|--------|-------|
| 1000   | 0.2 ms  | 0.1 ms  | 0.05 ms  | < 0.01 ms | < 0.01 ms |
| 5000   | 5.5 ms  | 2.8 ms   | 1.2 ms | 0.08 ms | 0.12 ms |
| 10000  | 22 ms  | 11 ms  | 4.8 ms   | 0.18 ms  | 0.25 ms |
| 50000  | 550 ms | 275 ms  | 120 ms  | 0.95 ms  | 1.4 ms |
| 100000 | 2.2 s  | 1.1 s | 0.48 s  | 2.1 ms  | 3.1 ms |
| 500000 | 約 55 s | 約 28 s | 約 12 s | 11.5 ms  | 16.5 ms | 

在實驗數據分析中，發現到各排序演算法的執行時間隨資料量增長的趨勢，均符合其時間複雜度之理論預測。以氣泡排序、選擇排序及插入排序為例，當資料量由 1,000 筆增加至 5,000 筆時，其運行時間約增加 25 倍，這與其時間複雜度的理論預期完全吻合。其次，針對合併排序與堆積排序，當資料量從 10,000 筆增加至 100,000 筆時，執行時間呈現約 10 餘倍的增長。數據經過對照後，整體實驗數據與理論模型相似，結果具合理性。

## 五、心得感想

經過這次的研究，我對於各種資料的排序演算法有著更多的認識，像是各種演算法會受到排序方法不同，排序所需的時間也會有所不同，像是明顯較為直觀的氣泡排序法相較於更複雜一些的合併排序所耗費的時間相對的會更多。此外資料量與執行時間的增長關係，遵循著時間複雜度的理論預測，讓我們可以大概預測運算的所需時間。此次研究也讓我知道在不同情境下應該選擇最適合的演算法，而不是單純追求最快的演算法。因為各個演算法優勢也都不同，並無好壞之分，必須根據資料量級或是系統環境下去選擇，使用最適合的方案，才能最快達成任務。
