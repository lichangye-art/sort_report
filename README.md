## 一、 前言
在資訊科學領域中，排序（Sorting）是資料處理最基礎且核心的環節。選擇合適的排序演算法，不僅能優化系統的執行效率，更能有效節省運算資源。本報告將針對氣泡、選擇、插入、合併及堆積五種經典演算法進行深入探討。

## 二、 演算法原理與複雜度分析

### 1. 氣泡排序法 (Bubble Sort)
**原理簡述：**
氣泡排序是一種直觀的交換排序法。它重複地走訪要排序的數列，一次比較兩個相鄰的元素，如果順序錯誤（如較大的在左側）就進行交換。重複此過程直到沒有任何元素需要交換。其特徵是每一輪排序後，該輪最大的元素會像「氣泡」一樣浮動到數列的最末端。

**複雜度分析：**
- **平均/最差時間複雜度：** $O(n^2)$
- **最佳時間複雜度：** $O(n)$（當數列已排序時）
- **空間複雜度：** $O(1)$

### 2. 選擇排序法 (Selection Sort)
**原理簡述：**
選擇排序的核心思想是「在未排序區域中尋找極值」。演算法會將數列分為「已排序」與「未排序」兩部分。每一輪都會從未排序區域中找出最小值，並將其放置在已排序區域的末尾。與氣泡排序不同，選擇排序每一輪只進行一次交換。

**複雜度分析：**
- **平均/最差時間複雜度：** $O(n^2)$
- **最佳時間複雜度：** $O(n^2)$
- **空間複雜度：** $O(1)$

### 3. 插入排序法 (Insertion Sort)
**原理簡述：**
插入排序模擬人類整理撲克牌的方式。它將數列分為已排序（左側）與未排序（右側）兩部分。逐一取回未排序區域的元素，在已排序區域中從後往前掃描，找到合適的位置後「插入」。此方法對於處理「幾乎已排好序」的資料具有極高的效率。

**複雜度分析：**
- **平均/最差時間複雜度：** $O(n^2)$
- **最佳時間複雜度：** $O(n)$
- **空間複雜度：** $O(1)$

### 4. 合併排序法 (Merge Sort)
**原理簡述：**
合併排序採用「分治法 (Divide and Conquer)」的策略。先將數列不斷對半分切，直到每個子序列只剩一個元素；接著，再將這些有序的子序列兩兩合併，合併過程中確保新的序列依然有序，最終重組成完整的排序數列。

**複雜度分析：**
- **平均/最佳/最差時間複雜度：** $O(n \log n)$
- **空間複雜度：** $O(n)$（需要額外空間暫存合併後的序列）

### 5. 堆積排序法 (Heap Sort)
**原理簡述：**
堆積排序利用了「堆積 (Heap)」這種半有序的資料結構。首先將原始數列建立成一個最大堆積，此時根節點即為最大值。接著將根節點與最後一個節點交換，並移除該最大值，隨後對剩餘結構進行調整使其重新符合堆積定義。重複此動作直到堆積清空。

**複雜度分析：**
- **平均/最佳/最差時間複雜度：** $O(n \log n)$
- **空間複雜度：** $O(1)$

## 三、 綜合比較與結論

這五種演算法各具優勢與限制：

- **基礎演算法 ($O(n^2)$ 級別)：** 氣泡、選擇與插入排序實作簡單，適合小規模資料。其中 **插入排序** 在處理局部有序資料時表現最為出色。
- **高效演算法 ($O(n \log n)$ 級別)：** 合併排序與堆積排序在處理大規模資料時更具效能優勢。**合併排序** 具有高度穩定性，而 **堆積排序** 則在空間利用上最為節省。

在實際應用中，通常會根據資料量大小、記憶體限制以及穩定性需求，靈活選擇最契合的排序策略。
